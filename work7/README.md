## 二、实验原理

### 2\.1 质点‑弹簧模型（Mass‑Spring Model）

将布料离散为网格质点，质点之间通过结构弹簧连接，通过胡克定律计算弹簧弹力，叠加阻尼力、重力完成物理模拟。

1. 弹簧弹力公式
$f_{a} = -k_{s} (|x_a - x_b| - l) \frac{x_a - x_b}{|x_a - x_b|}$
$k_s$：劲度系数；$l$：弹簧原长；$x_a,x_b$：相连两质点位置。

2. 阻尼力公式（抑制能量累积，防止爆炸）
$f_{d} = -k_{d} v_{a}$
$k_d$：阻尼系数；$v_a$：质点当前速度。

3. 牛顿第二定律：$a=F/m$，通过加速度更新质点速度、位置。

### 2\.2 三种数值积分方法

1. 显式欧拉（Explicit Euler）
使用当前时刻状态计算下一时刻，稳定性差、易爆炸：
$x_{t+1} = x_{t} + v_{t} \Delta t$
$v_{t+1} = v_{t} + a_{t} \Delta t$

2. 半隐式欧拉（Semi‑Implicit / 辛欧拉）
先更新速度，再用新速度更新位置，稳定性更好，图形学最常用：
$v_{t+1} = v_{t} + a_{t} \Delta t$
$x_{t+1} = x_{t} + v_{t+1} \Delta t$

3. 隐式欧拉（Implicit Euler）
使用未来状态求解，无条件稳定，本实验采用定点迭代近似求解：
$v_{t+1} = v_{t} + a_{t+1} \Delta t$
$x_{t+1} = x_{t} + v_{t+1} \Delta t$

## 五、代码整体逻辑与实验任务实现

### 任务 1：场景初始化（GPU 状态同步）

设置布料网格大小（20×20），初始化质点位置、速度、受力数组、弹簧拓扑；
严格拆分多个独立 @ti\.kernel：位置初始化 Kernel、弹簧初始化 Kernel、渲染索引初始化 Kernel；
Python 端按固定顺序调用 Kernel，保证 GPU 多线程下数据同步，避免读写冲突。

### 任务 2：力学计算与防爆处理

编写 @ti\.func compute\_forces\_on \(\)：统一计算重力、阻尼力、结构弹簧弹力；
使用 ti\.atomic\_add 保证多线程并行累加受力不冲突；
编写 @ti\.func clamp\_velocity \(\)：钳制质点最大速度，防止显式欧拉出现数值爆炸发散；
将两个函数声明为 ti\.func，编译时内联进 Kernel，降低 GPU 函数调用开销。

### 任务 3：积分求解器实现（核心）

分别实现三个独立 @ti\.kernel：
step\_explicit \(\)、step\_semi\_implicit \(\)、step\_implicit\_iter \(\)；
架构优化：受力计算、速度更新、位置更新合并在同一个 Kernel 内，减少每帧 Kernel 启动次数，提升 GPU 性能。

### 任务 4：渲染与 GGUI 交互

使用 ti\.ui\.Window 构建 3D 布料渲染场景；
通过 window\.GUI 添加交互按钮，支持实时切换三种积分器、暂停、重置；
实时观察不同积分方法下布料下垂、摆动、稳定程度的差异。

### 三种积分器稳定性对比

显式欧拉：相同阻尼下极易发散，布料剧烈抖动、飞散爆炸，稳定性最差；
半隐式欧拉：模拟稳定，布料自然下垂摆动，不易爆炸，图形学最常用；
隐式欧拉：无条件稳定，阻尼更大、收敛更快，布料运动偏僵硬，计算成本更高。

### 参数调节现象

阻尼系数较小：布料摆动幅度大、回弹明显；
阻尼系数较大：布料快速稳定、几乎无回弹；
未做速度钳制：显式欧拉直接数值爆炸，质点飞出场景；开启钳制后模拟正常。

### GPU 优化效果

ti\.func 内联后无额外函数调用开销，帧率更高；
合并 Kernel 大幅减少 CPU‑GPU 通信，模拟流畅无卡顿；
多 Kernel 分步初始化保证 GPU 并行状态正确，无数据错乱。

## 八、注意事项总结

多线程累加受力必须使用 ti\.atomic\_add，否则出现数据竞争、模拟错乱；
显式欧拉必须做速度钳制，否则极易出现数值爆炸；
GPU 并行下必须拆分初始化 Kernel 并顺序调用，保证状态同步；
ti\.func 仅能在 Kernel 内调用，会被编译内联，适合高频计算逻辑；
Kernel 启动开销较大，尽量合并计算步骤，减少单帧 Kernel 调用次数。
